# 离线强化学习

MASDiff 的策略提取阶段通过离线强化学习（Offline RL）完成。与在线 RL 不同，离线 RL 完全从已有的轨迹-奖励对 $(\tau, R)$ 中学习，无需与环境实时交互，从而保证训练稳定性。

## 核心思想

探索与利用彻底分离：

- **探索**：由进化策略在轨迹空间完成，产生高质量的 $(\tau, R)$ 数据集。
- **利用**：离线 RL 从这些数据集中稳定地提取策略 $\pi$。

这一分离避免了传统在线 MARL 中因非平稳环境导致的梯度冲突与训练崩溃。

## 各场景的离线 RL 实现

| 场景 | 策略架构 | 离线 RL 方式 |
|------|---------|-------------|
| 交通仿真（SUMO）| Hybrid A\* + Q-learning | Q 值回归（MSE），以生成奖励为目标 |
| SMAC | 独立 DQN（每单位一个网络）| DQN 离线更新，$(s, a, r_\text{gen}, s')$ 四元组 |
| MPE | 独立策略网络 | 离线策略梯度 |

## 训练流程

给定轨迹-奖励对 $(\tau_j, R_j)$：

1. **构建训练集**：将 $R_j[t, i]$（扩散模型生成的奖励）填入时间步 $t$ 对应的转移四元组，替换原始占位奖励。

2. **独立策略训练**：每个智能体 $i$ 独立优化自身策略：

$$\min_{\pi_i} \sum_{t} \left( Q_{\pi_i}(s_{i,t}, a_{i,t}) - r_{i,t}^{\text{gen}} \right)^2$$

3. **多轮迭代**：在同一批数据上训练若干 epoch，直至 Q 值收敛。

## 为什么不用在线 RL？

```{admonition} 在线 RL 的问题
:class: warning

在多智能体在线 RL 中：

- 所有智能体同时更新策略，导致环境非平稳（每个智能体看到的"环境"随其他智能体策略变化）。
- 探索（尝试新动作）和利用（更新策略）相互干扰，造成梯度冲突和训练不稳定。
- 难以在单次在线交互中同时优化宏观目标和个体奖励。
```

离线 RL 将这些问题彻底隔离：策略只在固定数据集上更新，环境状态确定性地由轨迹给出，训练过程完全可重现。

## 可扩展性

由于各智能体策略独立训练，整个离线 RL 阶段天然支持并行化：

```
for each agent i in parallel:
    train π_i on {(s_{i,t}, a_{i,t}, r_{i,t}^gen, s_{i,t+1})}
```

实验中 1557 辆车（智能体）的策略可在 4× RTX 4090 上并行训练，每次迭代耗时约 273 秒（Grid Nav 500 agents 基准）。
